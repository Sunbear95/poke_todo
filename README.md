# Poke Todo

할 일과 습관 완료를 캐릭터 만남, 포획, 컬렉션 보상으로 연결한 Expo 기반 모바일 MVP입니다.

일반적인 TODO 앱의 낮은 사용 마찰은 유지하면서, 하루의 행동이 눈에 보이는 보상으로 이어지는 경험을 검증합니다.

## 핵심 흐름

1. 오늘의 할 일이나 습관을 완료합니다.
2. 완료 상태와 습관 콤보를 바탕으로 만날 수 있는 캐릭터 후보를 확인합니다.
3. 하루 마감을 시작하면 완료 항목마다 하나의 만남이 생성됩니다.
4. 각 만남에서 포획을 시도하고 성공 또는 실패 결과를 확인합니다.
5. 포획한 캐릭터는 컬렉션에 추가됩니다.

같은 캐릭터를 반복해서 만나면 포획 확률이 올라가며, 습관 콤보는 선호 캐릭터의 출현 확률에 반영됩니다.

## 주요 기능

- 오늘의 할 일과 습관 확인 및 완료 처리
- 새로운 할 일과 습관 추가
- 완료 항목 기반 후보 캐릭터 미리보기
- 습관 콤보와 선호 캐릭터 설정
- 하루 마감 및 캐릭터 포획
- 반복 만남에 따른 포획 확률 증가
- 포획한 캐릭터 컬렉션 조회
- 마감 후 오늘의 기록 변경 방지

## 실행 방법

### 요구 사항

- Node.js 22 이상
- npm
- 모바일 실행 시 Expo Go 또는 Android/iOS 개발 환경

### 설치 및 실행

```bash
npm install
npm run dev
```

터미널에 표시되는 QR 코드를 Expo Go로 스캔하거나 다음 명령으로 플랫폼을 지정할 수 있습니다.

```bash
npm run android
npm run ios
```

같은 네트워크에서 실기기로 실행하려면 다음 명령을 사용할 수 있습니다.

```bash
npm run mobile
```

## 검증

```bash
npm test
npm run smoke
npm run build
```

- `npm test`: 도메인 모델과 핵심 보상 흐름 테스트
- `npm run smoke`: 완료부터 포획까지의 최소 시나리오 검증
- `npm run build`: 필수 프로젝트 파일 정적 검사

## MVP 범위

현재 버전은 핵심 보상 루프 검증에 집중한 로컬 데모입니다. 다음 기능은 의도적으로 제외했습니다.

- 사진 또는 증거 업로드
- AI 기반 완료 판정
- 포켓볼, 재화, 상점과 인벤토리
- 배틀과 소셜 기능

## 알려진 제한 사항

- 앱을 재시작하면 로컬 상태가 초기화됩니다.
- 날짜 전환과 장기 반복 습관 관리는 아직 지원하지 않습니다.
- 캐릭터 이미지는 외부 네트워크 리소스에 의존합니다.
- 공개 배포 전 오리지널 캐릭터 에셋과 실제 기기 QA가 필요합니다.

더 자세한 구현 및 검증 내용은 [`docs/assignment/README.md`](docs/assignment/README.md)에서 확인할 수 있습니다.
