# Lean Canvas

## 1. Key Partnerships

- 초기 사용자: 학생, 개인 프로젝트 진행자, 운동/공부 습관을 만들고 싶은 사용자
- 테스트 파트너: 수업 동료, 생산성 앱을 자주 써 본 사용자, 게임형 앱에 익숙한 사용자
- 에셋 파트너 또는 제작자: 공개 배포 시 오리지널 캐릭터 일러스트와 사운드 제작 필요
- 플랫폼 파트너: Expo/React Native, 향후 App Store와 Google Play
- 커뮤니티 파트너: 습관 형성, 공부 인증, 생산성 커뮤니티

## 2. Key Activities

- 오늘의 할 일과 습관을 쉽게 입력하고 완료 처리하는 경험 설계
- 완료 항목을 캐릭터 만남과 포획 기회로 변환하는 보상 규칙 개선
- 실루엣 후보, 하루 마감, 포획, 컬렉션 화면의 사용성 테스트
- 반복 만남과 포획 확률 상승이 동기부여에 주는 영향 검증
- 공개 배포 전 오리지널 캐릭터 세계관과 에셋 제작
- 장기 사용 데이터를 바탕으로 습관 유지율과 재방문율 분석

## 3. Key Resources

- 핵심 도메인 모델: 체크리스트, 후보 선정, 하루 마감, 포획, 컬렉션 로직
- Expo / React Native 모바일 앱 셸
- 캐릭터 데이터와 실루엣/컬렉션 UI
- 사용자 피드백과 사용성 테스트 결과
- 오리지널 캐릭터 IP 또는 공개 배포 가능한 에셋
- 제품의 핵심 원칙: 낮은 마찰, 하루 마감 보상, 수집 진행감

## 4. Value Propositions

Poke Todo는 해야 할 일을 더 복잡하게 관리하게 만드는 앱이 아니라, 이미 완료한 작은 행동을 더 기분 좋게 기억하게 만드는 앱이다.

핵심 가치 제안은 다음과 같다.

- 체크 한 번이면 보상 진행이 생긴다.
- 하루 마감 때 완료 항목이 캐릭터 만남으로 바뀐다.
- 실루엣 후보가 오늘의 작은 기대감을 만든다.
- 포획 실패도 다음 확률 상승으로 이어져 손해감이 줄어든다.
- 컬렉션이 쌓이면서 사용자는 자신의 꾸준함을 시각적으로 확인한다.

## 5. Customer Relationships

- MVP 단계: 직접 관찰, 인터뷰, 짧은 사용성 테스트 중심
- 초기 출시 단계: 온보딩에서 "체크만 해도 충분하다"는 부담 없는 메시지 전달
- 재방문 유도: 하루 마감 리마인드, 후보 실루엣, 컬렉션 진행률
- 장기 관계: 시즌별 오리지널 캐릭터, 개인 습관 기록 요약, 가벼운 성취 회고
- 고객 신뢰: 인증 업로드를 강요하지 않고 로컬 우선 데이터를 기본으로 설계

## 6. Channels

- 수업 발표와 데모
- 개인 네트워크와 수업 동료 테스트
- Expo Go 기반 모바일 데모 링크
- 생산성/습관 형성 커뮤니티
- 향후 앱스토어, 구글플레이, Product Hunt, Reddit 등 초기 출시 채널

## 7. Customer Segments

초기 고객은 "할 일 관리는 하고 싶지만, 딱딱한 생산성 앱에는 쉽게 질리는 사람"이다.

우선순위가 높은 세그먼트는 다음과 같다.

- 매일 공부나 운동 습관을 만들고 싶은 학생
- 개인 프로젝트를 꾸준히 진행하고 싶은 사람
- TODO 앱을 자주 설치하지만 오래 쓰지 못하는 사용자
- 게임형 보상과 수집 경험에 익숙한 사용자
- 인증형 습관 앱은 부담스럽지만 가벼운 동기부여는 원하는 사용자

## 8. Cost Structure

MVP 단계에서는 서버 비용 없이 로컬 데모 중심으로 운영할 수 있다.

주요 비용은 다음과 같다.

- 제품 개발 시간
- 사용자 테스트와 인터뷰 시간
- 공개 배포용 오리지널 캐릭터 에셋 제작 비용
- 향후 모바일 앱 배포 계정 비용
- 장기적으로는 데이터 저장, 백업, 알림, 계정 기능을 위한 인프라 비용
- 마케팅과 커뮤니티 운영 비용

## 9. Revenue Streams

초기 MVP에서는 수익화보다 핵심 루프 검증이 우선이다. 이후 가능한 수익 모델은 다음과 같다.

- 프리미엄 구독: 추가 캐릭터 테마, 고급 통계, 백업/동기화
- 일회성 결제: 시즌 캐릭터 팩, 테마 팩
- 교육/스터디 그룹용 라이선스: 스터디 모임이나 수업 단위의 가벼운 습관 챌린지
- 브랜드 협업 캐릭터: 공개 배포 후 IP 문제가 없는 범위에서 협업 테마 제공

초기에는 하나의 수익 모델로 좁힌다면 "무료 기본 앱 + 선택형 프리미엄 테마/백업 구독"이 가장 자연스럽다.
