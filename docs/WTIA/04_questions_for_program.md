# 프로그램 중 묻고 싶은 질문

## 아이디어 검증

- 후보 실루엣과 하루 마감 포획 루프가 실제 재방문 동기로 충분한가?
- 사용자가 이 앱을 TODO 앱으로 인식하는 것이 좋은가, 아니면 습관 게임으로 인식하는 것이 좋은가?
- 초기 고객을 학생, 자기계발 사용자, 개인 프로젝트 진행자 중 어디로 좁히는 것이 가장 설득력 있는가?

## 제품 범위

- MVP에서 캐릭터 수집 외에 연속 기록, 통계, 리마인드 중 무엇을 먼저 추가해야 하는가?
- 포획 실패 후 확률 상승은 충분히 좋은 보상인가, 아니면 실패에도 작은 보상을 추가해야 하는가?
- 하루 마감이라는 구조가 사용자의 실제 루틴과 잘 맞는가?

## 시장과 차별화

- 일반 TODO 앱, 습관 추적 앱, 게임형 생산성 앱 사이에서 Poke Todo의 가장 강한 차별점은 무엇으로 설명해야 하는가?
- 포켓몬 스타일의 수집 문법을 오리지널 캐릭터로 바꿔도 투자자나 사용자가 즉시 이해할 수 있는가?
- "낮은 마찰"을 핵심 가치로 둘 때 기능을 얼마나 적게 유지해야 하는가?

## 피치덱과 발표

- 발표에서는 Problem-Solution 구조와 고객 여정 구조 중 어느 쪽이 더 설득력 있는가?
- 시장 규모를 생산성 앱 시장으로 잡아야 하는가, 습관 형성 앱 시장으로 잡아야 하는가?
- 현재 traction을 구현된 데모와 테스트로 설명해도 충분한가, 아니면 사용자 인터뷰 결과가 꼭 필요한가?

## 다음 액션

- 5-10명의 초기 사용자 테스트에서 반드시 물어봐야 할 질문은 무엇인가?
- 수익화는 구독, 테마 팩, 교육용 라이선스 중 무엇부터 가설로 잡는 것이 좋은가?
- 공개 배포 전 IP 리스크를 줄이기 위해 캐릭터 세계관을 어떻게 바꾸는 것이 자연스러운가?
