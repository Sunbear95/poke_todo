# 01. 프로젝트 개요

## 과제 주제

Poke Todo는 일반적인 TODO/습관 앱에 게임형 보상 루프를 결합한 앱이다. 사용자가 오늘의 할 일이나 습관을 완료하면, 하루 마감 단계에서 포켓몬 스타일의 캐릭터를 만나고 포획을 시도할 수 있다.

## 문제 정의

기존 TODO 앱은 해야 할 일을 정리하기에는 편하지만, 매일 다시 열고 싶게 만드는 감정적 보상이 약하다. 반대로 지나치게 복잡한 습관 앱이나 인증형 앱은 사진 업로드, 증명, 기록 작성처럼 사용자의 부담을 키운다.

이 과제는 “체크 한 번이면 충분한 낮은 마찰”과 “다시 돌아오고 싶게 만드는 보상”을 동시에 만족하는 MVP를 만드는 데 초점을 둔다.

## 핵심 아이디어

사용자는 오늘 할 일을 체크한다. 앱은 완료된 항목을 바탕으로 보상 후보 캐릭터를 실루엣으로 보여준다. 하루 마감을 시작하면 완료한 항목 하나당 하나의 만남이 생성되고, 사용자는 각 만남에서 포획을 시도한다. 성공하면 캐릭터가 컬렉션에 추가된다.

## MVP 목표

- 오늘의 할 일과 습관을 확인하고 완료 처리할 수 있다.
- 완료한 항목을 바탕으로 2-3개의 보상 후보 실루엣을 보여준다.
- 하루 마감 시 완료한 항목 수만큼 캐릭터 만남을 만든다.
- 사용자가 직접 포획을 시도하고 성공/실패 결과를 확인한다.
- 같은 캐릭터를 반복해서 만나면 포획 확률이 오른다.
- 포획에 성공한 캐릭터는 컬렉션 화면에서 확인할 수 있다.

## 의도적으로 제외한 범위

MVP에서는 할 일 인증 사진, AI 증명 분석, 포켓볼 소모, 재화, 상점, 인벤토리, 배틀, 소셜 기능을 제외했다. 첫 버전의 목적은 큰 게임 시스템을 만드는 것이 아니라, “할 일 완료가 보상 경험으로 이어지는가”를 검증하는 것이다.

## 현재 구현 형태

현재 프로젝트는 공통 도메인 모델을 중심으로 하나의 모바일 앱 셸을 제공한다.

| 영역 | 설명 |
|---|---|
| `src/model.mjs` | 체크리스트, 후보 선정, 하루 마감, 포획, 컬렉션 로직 |
| `src/demoData.mjs` | 데모용 캐릭터와 할 일 데이터 |
| `App.js` | Expo / React Native 기반 모바일 앱 셸 |
| `test/model.test.mjs` | 핵심 모델 동작 테스트 |

## 기대 결과

과제의 결과물은 사용자가 단순히 할 일을 완료하는 데서 끝나지 않고, 하루의 행동이 작은 보상 이벤트와 컬렉션 진행으로 이어지는 경험을 보여주는 것이다.
