# 04. 검증 및 범위

## 검증 목표

검증의 목표는 앱이 “할 일 완료 -> 후보 실루엣 -> 하루 마감 -> 포획 -> 컬렉션”이라는 MVP 핵심 루프를 안정적으로 수행하는지 확인하는 것이다.

## 실행 명령

```bash
npm install
npm test
npm run smoke
npm run build
npm run dev
npm run mobile
```

## 테스트가 확인하는 핵심 동작

- 새 할 일을 추가하면 미완료 할 일로 목록에 들어간다.
- 잘못된 제목이나 종류는 검증 오류로 처리된다.
- 할 일을 체크/해제하면 완료 개수가 갱신된다.
- 습관을 완료하면 콤보가 증가한다.
- 후보 캐릭터는 2-3개로 제한되고 실루엣/예고 상태로 표시된다.
- 완료 항목이 없으면 후보 보상이 나오지 않는다.
- 하루 마감은 완료 항목 수만큼 만남을 만든다.
- 미완료 항목은 만남을 만들지 않는다.
- 같은 캐릭터 반복 만남은 포획 확률을 올린다.
- 포획 성공 시 컬렉션에 추가된다.
- 포획 실패 시에도 앱이 안전하게 상태를 유지한다.
- 증명 업로드, AI 판정, 포켓볼, 재화, 상점, 인벤토리 상태가 MVP에 섞이지 않는다.

| 수용 기준 | 현재 상태 |
|---|---|
| 오늘의 체크리스트가 시작된다 | 통과 |
| 사용자가 항목을 완료 처리할 수 있다 | 통과 |
| 후보 실루엣이 표시된다 | 통과 |
| 하루 마감이 완료 항목 기반 만남을 만든다 | 통과 |
| 사용자가 포획을 시도할 수 있다 | 통과 |
| 반복 만남으로 포획 확률이 오른다 | 통과 |
| 성공한 포획이 컬렉션에 표시된다 | 통과 |
| MVP 제외 기능이 없다 | 통과 |
| 화면 상태가 이해 가능하다 | 통과, 추가 수동 리뷰 권장 |
| 문서화된 사용 명령이 동작한다 | 통과 |

## 남아 있는 한계

현재 결과물은 MVP 검증에 초점을 둔 로컬 데모다. 실제 배포 앱으로 발전시키려면 다음 항목을 추가로 결정해야 한다.

- 공개 배포용 오리지널 캐릭터 에셋으로 교체
- 실제 모바일 앱 배포 방식 결정
- 자동화된 브라우저 E2E 테스트 확대
- 장기 사용을 위한 날짜 전환, 반복 습관 관리, 데이터 백업 정책 보강
- 접근성 및 실제 기기 수동 QA 확대

## 결론

현재 구현은 과제의 핵심 목표인 낮은 마찰의 TODO/습관 완료와 게임형 보상 루프를 설명하고 시연하기에 충분한 MVP 상태다. 특히 핵심 규칙은 모델 테스트로 검증되어 있으며, 웹 데모와 Expo 셸은 같은 모델을 공유하도록 구성되어 있다.
