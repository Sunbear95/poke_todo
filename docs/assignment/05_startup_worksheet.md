# 05. 창업 아이디어 워크시트

## 아이디어 이름

Poke Todo

## 한 줄 설명

Poke Todo는 매일의 할 일과 습관 완료를 캐릭터 만남, 포획 시도, 컬렉션 진행으로 연결하는 낮은 마찰의 TODO/습관 관리 앱이다.

## WHY 문장

Poke Todo는 사용자가 할 일을 기록하는 부담을 늘리지 않으면서도 매일 다시 돌아오고 싶게 만들기 위해, 체크 한 번을 작은 보상 경험과 수집 진행으로 바꾸기 위해 만든다.

## 1. 해결하려는 문제는?

기존 TODO 앱은 해야 할 일을 정리하고 체크하는 데는 편하지만, 완료 후의 감정적 보상이 약하다. 사용자는 목록을 지우는 순간에는 잠깐 만족하지만, 다음 날 다시 앱을 열 이유는 부족하다.

반대로 많은 습관 앱은 인증 사진, 긴 기록, 통계 확인, 복잡한 레벨 시스템처럼 사용자의 입력 부담을 키운다. 꾸준함을 만들기 위한 도구가 오히려 새로운 숙제가 되는 문제가 있다.

Poke Todo가 해결하려는 핵심 문제는 다음과 같다.

- 할 일 앱은 유용하지만 재미와 기대감이 약하다.
- 습관 앱은 동기부여를 주려다가 기록 부담을 키우기 쉽다.
- 사용자는 매일의 작은 행동이 축적되고 있다는 감각을 원한다.
- 완료한 행동이 눈에 보이는 보상으로 이어지지 않으면 재방문 동기가 약해진다.

## 2. 고객 페인포인트는?

주요 고객은 할 일과 습관을 관리하고 싶지만, 너무 진지하거나 복잡한 생산성 도구에는 쉽게 지치는 사람이다. 특히 학생, 개인 프로젝트를 진행하는 사람, 운동이나 공부 습관을 만들고 싶은 사용자가 초기 고객이 될 수 있다.

페인포인트는 다음과 같다.

- TODO 앱을 설치해도 며칠 뒤에는 앱을 열지 않게 된다.
- 습관을 인증하거나 긴 일지를 쓰는 과정이 번거롭다.
- 완료한 행동이 쌓이는 느낌이 약해서 성취감이 오래가지 않는다.
- 게임형 앱은 재미있지만, 재화와 상점, 전투 같은 부가 시스템이 많아 피로할 수 있다.
- 작은 목표를 완료했을 때 즉각적이고 가벼운 보상을 받고 싶다.

## 3. 해결 접근법은?

Poke Todo는 TODO 앱의 기본 행동을 바꾸지 않는다. 사용자는 평소처럼 오늘의 할 일이나 습관을 체크한다. 대신 앱은 완료된 항목을 하루 마감 시점에 보상 만남으로 바꾼다.

핵심 접근법은 다음과 같다.

- 체크 한 번이면 충분하다. 사진 업로드나 AI 인증은 MVP에서 제외한다.
- 완료한 항목 1개는 캐릭터 만남 1개로 이어진다.
- 하루 중에는 보상 후보를 실루엣으로 보여주어 기대감을 만든다.
- 하루 마감에서 사용자가 직접 포획을 시도하게 해 보상 순간을 능동적으로 만든다.
- 포획 실패도 완전한 손해가 아니라, 다음 만남의 포획 확률 상승으로 이어지게 한다.
- 성공한 캐릭터는 컬렉션에 저장되어 장기적인 진행감을 만든다.

## 4. 왜 지금이 적기인가?

사용자는 생산성 도구를 계속 찾지만, 단순한 체크리스트만으로는 장기 사용을 유지하기 어렵다. 동시에 너무 복잡한 자기관리 앱은 매일 쓰기 부담스럽다. 그래서 낮은 입력 부담과 즉각적인 보상을 함께 제공하는 제품에 기회가 있다.

지금 이 아이디어가 적기인 이유는 다음과 같다.

- 습관 형성, 자기관리, 게임화된 생산성에 대한 수요가 계속 있다.
- 모바일 사용자는 짧고 반복적인 보상 루프에 익숙하다.
- 복잡한 AI 인증 없이도 MVP 핵심 가치를 로컬 앱으로 빠르게 검증할 수 있다.
- 포켓몬 스타일의 수집 경험은 사용자가 직관적으로 이해하는 보상 문법이다.
- 초기 버전은 서버, 인증, 결제 없이도 작동하므로 빠르게 테스트할 수 있다.

## 5. 왜 아직 해결되지 않았나?

많은 TODO 앱은 작업 관리에 집중하고, 많은 게임형 앱은 게임 시스템에 집중한다. 그 사이에서 "체크리스트는 그대로 단순하게 유지하면서, 완료 후의 보상만 충분히 즐겁게 만드는" 제품은 상대적으로 덜 명확하게 다뤄졌다.

해결이 어려웠던 이유는 다음과 같다.

- TODO 앱은 기능이 많아질수록 본래 장점인 단순함을 잃기 쉽다.
- 게임형 보상을 넣으면 재화, 상점, 레벨, 배틀 같은 시스템으로 범위가 커지기 쉽다.
- 인증 기반 습관 앱은 신뢰성을 높이지만 사용자의 일일 부담도 커진다.
- 보상 경험이 너무 자동적이면 재미가 약하고, 너무 복잡하면 습관 관리와 충돌한다.

Poke Todo는 MVP에서 의도적으로 많은 기능을 제외하고, "체크 -> 후보 실루엣 -> 하루 마감 -> 포획 -> 컬렉션" 루프 하나만 검증한다.

## 6. 왜 내가 이 아이디어에 적임자인가?

SHON은 이 과제를 정리하는 데 도움을 준 사람이고, 실제 아이디어의 문제의식과 개발은 내가 혼자 진행하고 있다. 이 아이디어는 내가 직접 여러 계획을 세우고 실패해 본 경험에서 출발했다. 계획을 세우는 것 자체보다, 다시 돌아오고 싶게 만드는 동기와 보상 루프가 부족하다는 점이 문제의 핵심이라고 느꼈다.

또한 나는 게임을 많이 하기 때문에, 사용자가 반복해서 돌아오게 만드는 보상 구조, 수집 욕구, 실패 후 재도전 동기, 확률과 기대감의 감각을 생활 속에서 익숙하게 이해하고 있다. 그래서 TODO 앱을 단순한 생산성 도구가 아니라, 매일의 행동을 작고 즐거운 게임 루프로 바꾸는 방식으로 바라볼 수 있다.

적합한 이유는 다음과 같다.

- 스스로 계획을 많이 세우고 실패해 본 경험이 있어, 사용자가 왜 TODO 앱을 오래 쓰지 못하는지 개인적으로 이해한다.
- 게임을 많이 해 왔기 때문에 보상, 수집, 랜덤성, 재도전 동기 같은 요소가 왜 사람을 붙잡는지 감각적으로 알고 있다.
- 문제를 "계획을 더 잘 세우는 앱"이 아니라 "계획을 다시 실행하게 만드는 보상 루프"로 정의했다.
- 개발을 혼자 진행하고 있어 MVP 범위를 작게 잡고 빠르게 검증할 수 있다.
- 인증, 재화, 상점, 배틀 같은 유혹적인 확장 기능을 MVP에서 제외하고 핵심 루프에 집중했다.

## 7. 1차 검증 가설

Poke Todo의 초기 검증은 제품이 완벽한지보다, 핵심 보상 루프가 실제로 다시 열고 싶은 느낌을 주는지에 집중한다.

검증할 가설은 다음과 같다.

| 가설 | 확인 방법 |
|---|---|
| 체크 후 실루엣 후보가 보이면 사용자가 보상을 기대한다 | 데모 사용 중 후보 영역을 인지하는지 관찰 |
| 하루 마감 포획 시도가 일반 완료 알림보다 더 기억에 남는다 | 사용 후 인터뷰에서 가장 기억나는 순간 질문 |
| 포획 실패 후 확률 상승이 재시도 동기를 만든다 | 실패 경험 후 다음 사용 의향 확인 |
| 컬렉션 화면이 장기 사용 이유가 된다 | 사용자가 수집 상태를 다시 확인하는지 관찰 |
| 인증 없이 체크만 하는 방식이 부담을 줄인다 | 기존 습관 앱과 비교해 입력 부담 질문 |

## 8. 백업 아이디어

메인 아이디어는 Poke Todo다. 필요하다면 다음과 같은 인접 아이디어로 좁히거나 확장할 수 있다.

| 후보 | 설명 | 메인 아이디어와의 관계 |
|---|---|---|
| Study Catch | 공부 세션 완료를 캐릭터 포획으로 연결하는 학습 전용 앱 | 고객 세그먼트를 학생으로 좁힌 버전 |
| Habit Garden | 습관 완료가 식물 성장과 수집으로 이어지는 앱 | 포켓몬 스타일 IP 리스크를 줄인 대체 테마 |
| Quest Checklist | 오늘의 할 일을 퀘스트로 바꾸고 하루 마감에서 보상을 지급하는 앱 | 포획보다 퀘스트 판타지에 가까운 버전 |

현재는 Poke Todo가 가장 직접적이고 시연 가능성이 높으므로 메인 아이디어로 유지한다.
