# 02. 기능 흐름

## 전체 사용자 흐름

```txt
오늘의 기록 열기
  -> 할 일/습관 확인
  -> 완료 항목 체크
  -> 보상 후보 실루엣 확인
  -> 하루 마감 시작
  -> 완료 항목별 만남 생성
  -> 포획 시도
  -> 성공/실패 확인
  -> 컬렉션 확인
```

## 1. 오늘의 기록

앱의 첫 화면은 오늘의 할 일과 습관 목록이다. 사용자는 각 항목을 체크해서 완료 상태로 바꿀 수 있다.

현재 데모에는 건강, 운동, 공부, 취미처럼 일상적인 카테고리가 들어가 있으며, 앱에서 직접 새 할 일을 추가할 수도 있다.

## 2. 할 일 상세

할 일을 선택하면 상세 화면에서 목표, 메모, 카테고리, 보상 XP를 볼 수 있다. 습관 항목인 경우에는 연속 수행 일수인 콤보가 표시된다.

상세 화면에는 “만날 수 있는 몬스터” 후보가 실루엣 형태로 표시된다. 이 후보는 보상을 미리 확정하는 것이 아니라, 사용자가 기대감을 가질 수 있도록 보여주는 예고다.

## 3. 습관 콤보와 선호 후보

습관 항목에서는 사용자가 후보 캐릭터를 선택할 수 있다. 콤보 일수가 높을수록 선택한 후보의 등장 가중치가 증가한다.

이 구조는 반복 습관을 유지하는 사용자에게 “내가 원하는 보상에 가까워진다”는 느낌을 주기 위한 장치다.

## 4. 하루 마감

사용자가 하루 마감을 누르면 완료된 할 일만 보상 만남으로 변환된다. MVP 규칙은 단순하다.

```txt
완료한 항목 1개 = 만남 1개
미완료 항목 = 만남 없음
```

완료 항목이 없다면 앱은 빈 상태를 안전하게 보여준다.

## 5. 포획 시도

각 만남에는 포획 확률이 있다. 사용자가 포획 시도를 누르면 성공 또는 실패 결과가 나온다.

포획에는 포켓볼, 재화, 상점, 인벤토리가 필요하지 않다. 이 결정은 MVP의 낮은 마찰 원칙을 지키기 위한 것이다.

## 6. 반복 만남 보정

처음 만나는 캐릭터의 기본 포획 확률은 20%다. 같은 캐릭터를 다시 만날수록 포획 확률이 15%씩 올라가며, 최대 85%까지 증가한다.

```txt
0회 이전 만남: 20%
1회 이전 만남: 35%
2회 이전 만남: 50%
최대값: 85%
```

실패가 완전한 손해로 끝나지 않고 다음 기회의 확률 상승으로 이어지도록 설계했다.

## 7. 컬렉션

포획에 성공한 캐릭터는 컬렉션 화면에 표시된다. 아직 잡지 못한 캐릭터는 실루엣 또는 미발견 상태로 남아, 다음 사용 동기를 만든다.

## 핵심 경험

이 앱의 핵심은 할 일 앱의 기본 행동을 복잡하게 만들지 않으면서, 완료 후에 작고 명확한 보상 순간을 제공하는 것이다.
